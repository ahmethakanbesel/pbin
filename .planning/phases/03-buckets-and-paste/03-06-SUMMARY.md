## Decisions Made

- Go 1.22 `ServeMux` panics at startup when two patterns overlap without one being strictly more specific. This happened with `GET /b/{slug}` vs `GET /{slug}/info` (both match `/b/info`), and with `GET /delete/{slug}/{secret}` vs `GET /{slug}/file/{storageKey}` (both match `/delete/file/x`). Resolved by consolidating all GET routes into a single `GET /` subtree catch-all that dispatches manually based on path segment count and fixed literals.
- Paste raw URL changed from `/raw/{slug}` to `/{slug}/raw` for consistency with `/{slug}/info` and to eliminate the conflict with `/b/{slug}`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Go 1.22 mux pattern conflict panics at server startup**
- **Found during:** Task 2 (checkpoint verification — server startup)
- **Issue:** Go 1.22's `net/http.ServeMux` panics at registration time when two patterns overlap but neither is strictly more specific. Occurring for: `GET /b/{slug}` vs `GET /{slug}/info` (conflict at `/b/info`); `GET /delete/{slug}/{secret}` vs `GET /{slug}/file/{storageKey}` (conflict at `/delete/file/x`); `GET /raw/{slug}` vs `GET /{slug}/info` (conflict at `/raw/info`).
- **Fix:** Replaced all individual GET patterns with a single `GET /` catch-all handler. Manual path parsing with `strings.Split` dispatches to the correct handler arm. `r.SetPathValue()` is used to populate named path values before delegating to domain handlers. Paste raw moved from `/raw/{slug}` to `/{slug}/raw`.

The single catch-all approach cleanly resolves all of them.
