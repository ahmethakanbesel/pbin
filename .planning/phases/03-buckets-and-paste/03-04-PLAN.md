**Upload — handles multipart with multiple file fields:**
- Call `securityHeaders(w)` first
- Apply `http.MaxBytesReader` before parsing
- Call `r.ParseMultipartForm(32 << 20)` and `defer r.MultipartForm.RemoveAll()`
- Iterate over `r.MultipartForm.File["file"]` (note: `r.MultipartForm.File` is `map[string][]*multipart.FileHeader` — use `r.MultipartForm.File["file"]` slice for multiple files)
- For each file header: open it, peek 512 bytes to detect MIME type (same `http.DetectContentType` pattern as file handler), force `application/octet-stream` for dangerous MIME types, combine peeked bytes with remainder via `io.MultiReader`
- Parse `expiry` (default `"1d"`), `password`, `one_use == "1"` from form values
- Build `bucket.CreateRequest{Files: []bucket.FileInput{...}, ...}` and call `h.svc.Create`
- Return JSON response:
```go
type bucketUploadResponse struct {
    URL       string           `json:"url"`
    DeleteURL string           `json:"delete_url"`
    ExpiresAt *string          `json:"expires_at"` // RFC3339 or null
    FileCount int              `json:"file_count"`
    Files     []bucket.FileInfo `json:"files"`
}
```
- Return 201 on success

**View — GET /{slug} (bucket) — HTML page:**
- Call `securityHeaders(w)`
- Read password from `X-Password` header or `?password=` query param
- Call `h.svc.GetMeta(ctx, slug, password)`
- On `ErrWrongPassword`: serve password form (same `servePasswordForm` function, reuse it from handler package)
- On `ErrExpired`: 410 Gone JSON
- On `ErrNotFound`: 404 JSON
- On success: render HTML page with:
  - Title: `{slug} — pbin`
  - Metadata bar: file count, expiry info
  - File list: each BucketFile with filename, human-readable size, no individual download link (ZIP download only)
  - Prominent "Download all as ZIP" button linking to `/{slug}/zip`
  - If password was provided: pass it as `?password=` on the ZIP link
  - CSP: `default-src 'none'; style-src 'unsafe-inline'; form-action 'self'`
  - Use inline CSS for a clean, minimal PsiTransfer-style layout

**DownloadZIP — GET /{slug}/zip:**
- Call `securityHeaders(w)`
- Read password from `X-Password` header or `?password=` query param
- Call `h.svc.StreamZIP(ctx, slug, password, w)`
- StreamZIP sets Content-Type and Content-Disposition headers internally and writes to w
- Handle errors: `ErrNotFound` → 404, `ErrExpired` → 410, `ErrAlreadyConsumed` → 410, `ErrWrongPassword` → 401 JSON or password form
- IMPORTANT: check errors BEFORE any bytes are written to w; call `h.svc.StreamZIP` only after all other checks, as it writes headers directly

**Delete — GET /delete/{slug}/{secret}:**
- Call `securityHeaders(w)`
- Extract `slug` and `secret` from path values
- Call `h.svc.Delete(ctx, slug, secret)`
- Return `{"deleted": true}` on 200 success
- `ErrNotFound` → 404, `ErrBadDeleteSecret` → 403

Note: `securityHeaders`, `writeJSON`, `writeError`, and `servePasswordForm` are defined in `file.go` in the same `handler` package — do NOT redefine them. Just call them directly.
