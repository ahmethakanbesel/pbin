Decision from 03-CONTEXT.md:
- Paste view: GET /{slug} — same URL pattern as files; differentiate by entity type lookup
- Bucket upload: POST /api/upload?type=bucket
- Individual bucket files: GET /{slug}/file/{storageKey} (LOCKED: individual files are downloadable separately)

Routing decisions:
- `POST /api/upload` — if `r.URL.Query().Get("type") == "bucket"` dispatch to bucketHandler.Upload, else fileHandler.Upload
- `GET /{slug}` — needs to look up which entity type owns the slug and route accordingly
- `GET /{slug}/zip` — bucketHandler.DownloadZIP (registered before GET /{slug} in Go 1.22 mux)
- `GET /{slug}/file/{storageKey}` — bucketHandler.DownloadFile (registered before GET /{slug})
- `GET /raw/{slug}` — pasteHandler.Raw
- `POST /api/paste` — pasteHandler.Create
- `GET /delete/{slug}/{secret}` — needs to dispatch to correct Delete handler based on entity type

Slug dispatch approach: Create a dispatcher handler that attempts lookup in files first, then buckets, then pastes, and delegates to the matching handler. This avoids adding a type column across tables.

3. Replace the existing route registrations with:
```go
mux.HandleFunc("GET /health", handler.Health)

// Upload: dispatch on ?type=bucket
mux.HandleFunc("POST /api/upload", func(w http.ResponseWriter, r *http.Request) {
    if r.URL.Query().Get("type") == "bucket" {
        bucketHandler.Upload(w, r)
    } else {
        fileHandler.Upload(w, r)
    }
})

// Paste creation
mux.HandleFunc("POST /api/paste", pasteHandler.Create)

// More-specific patterns before the wildcard (Go 1.22 requires this)
mux.HandleFunc("GET /{slug}/info", fileHandler.Info)
mux.HandleFunc("GET /{slug}/zip", bucketHandler.DownloadZIP)
mux.HandleFunc("GET /{slug}/file/{storageKey}", bucketHandler.DownloadFile)
mux.HandleFunc("GET /raw/{slug}", pasteHandler.Raw)

// Universal slug dispatch: files, buckets, pastes
mux.HandleFunc("GET /{slug}", func(w http.ResponseWriter, r *http.Request) {
    sl := r.PathValue("slug")

    // Try file — route to fileHandler for all "file exists" signals:
    // err == nil (no password, not expired), ErrWrongPassword, ErrExpired.
    // Only skip if ErrNotFound.
    if _, err := fileSvc.GetMeta(r.Context(), sl); err == nil ||
        errors.Is(err, file.ErrWrongPassword) || errors.Is(err, file.ErrExpired) {
        fileHandler.Serve(w, r)
        return
    }
    // Try bucket — route to bucketHandler for all "bucket exists" signals.
    if _, err := bucketSvc.GetMeta(r.Context(), sl, ""); err == nil ||
        errors.Is(err, bucket.ErrWrongPassword) || errors.Is(err, bucket.ErrExpired) {
        bucketHandler.View(w, r)
        return
    }
    // Fall through to paste handler — it handles ErrNotFound with a 404.
    pasteHandler.View(w, r)
})

// Universal delete dispatch
mux.HandleFunc("GET /delete/{slug}/{secret}", func(w http.ResponseWriter, r *http.Request) {
    sl := r.PathValue("slug")

    // Identify entity type by slug existence only (not by secret).
    // Use GetMeta with empty password — only care whether ErrNotFound comes back.
    if _, err := fileSvc.GetMeta(r.Context(), sl); !errors.Is(err, file.ErrNotFound) {
        fileHandler.Delete(w, r)
        return
    }
    if _, err := bucketSvc.GetMeta(r.Context(), sl, ""); !errors.Is(err, bucket.ErrNotFound) {
        bucketHandler.Delete(w, r)
        return
    }
    pasteHandler.Delete(w, r) // will return 404 if not found
})
```

4. The inline delete dispatcher calls `fileHandler.Delete` and `bucketHandler.Delete` which internally validate the delete secret — no secret validation happens in main.go. This is correct: the dispatcher's only job is routing.

5. Note: `writeJSON` is unexported from the handler package. Do NOT call it in main.go. The delete dispatcher delegates entirely to the respective handler's Delete method, so no JSON writing is needed in main.go.
