```go
// BasicAuth returns a middleware that enforces HTTP Basic Auth when cfg.Auth.Enabled is true.
// When disabled, the handler is called directly with no overhead.
// Credential comparison uses crypto/subtle.ConstantTimeCompare to prevent timing attacks.
func BasicAuth(cfg config.AuthConfig, next http.Handler) http.Handler
```

Implementation details:
- If `cfg.Auth.Enabled == false`, return `next` unchanged (zero overhead path).
- If enabled, return an `http.HandlerFunc` that:
  1. Calls `w.Header().Set("X-Content-Type-Options", "nosniff")` and `w.Header().Set("X-Frame-Options", "DENY")`.
  2. Reads username and password via `r.BasicAuth()`. If the call returns `ok == false`, respond with `WWW-Authenticate: Basic realm="pbin"` header, status 401, body `{"error":"authentication required"}` JSON. Return.
  3. Uses `subtle.ConstantTimeCompare([]byte(username), []byte(cfg.Auth.Username))` AND `subtle.ConstantTimeCompare([]byte(password), []byte(cfg.Auth.Password))`. Both must return 1. If either fails, same 401 response as step 2. Return.
  4. On success, call `next.ServeHTTP(w, r)`.

Imports needed: `crypto/subtle`, `encoding/json`, `net/http`, `github.com/ahmethakanbesel/pbin/internal/config`.

Do NOT import the handler package (circular import risk). Write the JSON response inline.

```go
type Cleanup struct {
    fileRepo   FileRepository
    bucketRepo BucketRepository
    pasteRepo  PasteRepository
    store      filestore.Backend
    interval   time.Duration
    stop       chan struct{}
}

func NewCleanup(
    fileRepo FileRepository,
    bucketRepo BucketRepository,
    pasteRepo PasteRepository,
    store filestore.Backend,
    interval time.Duration,
) *Cleanup

// Start launches the background cleanup goroutine. Call once at startup.
// The goroutine restarts itself on panic (defer/recover).
func (c *Cleanup) Start()

// Stop signals the cleanup goroutine to exit. Blocks until the goroutine acknowledges.
// Use a done chan struct{} for the acknowledgement.
func (c *Cleanup) Stop()
```

Cleanup loop logic (in a private `runLoop` method):

1. Wait for `c.interval` ticker OR `c.stop` signal.
2. On tick, call `sweep(ctx)` with a 60-second context timeout.
3. On stop, return (defers the done signal).

`sweep(ctx context.Context)` logic:

**Files:**
- Call `fileRepo.ListExpired(ctx)` — collect all expired `file.File` entities.
- Close the result immediately (ListExpired returns a slice, not rows — no open cursor).
- For each file: call `store.Delete(ctx, file.StorageKey)` then `fileRepo.Delete(ctx, file.Slug)`. Log errors but continue.
- Count deleted files.

**Buckets:**
- Call `bucketRepo.ListExpired(ctx)` — collect all expired `bucket.Bucket` entities.
- For each bucket: for each `bf` in `bucket.Files`, call `store.Delete(ctx, bf.StorageKey)`. Then call `bucketRepo.Delete(ctx, bucket.Slug)`. Log errors but continue.
- Count deleted buckets.

**Pastes:**
- Call `pasteRepo.ListExpired(ctx)` — collect all expired `paste.Paste` entities.
- For each paste: call `pasteRepo.Delete(ctx, paste.Slug)`. No on-disk data for pastes.
- Count deleted pastes.

After all three: `slog.Info("cleanup sweep complete", "files", nFiles, "buckets", nBuckets, "pastes", nPastes)`.

Panic recovery (wraps the entire goroutine body):
```go
go func() {
    defer func() {
        if r := recover(); r != nil {
            slog.Error("cleanup worker panicked, restarting", "panic", r)
            time.Sleep(5 * time.Second)
            c.runLoop()
        }
    }()
    c.runLoop()
}()
```

Note: The Bucket entity's `Files []BucketFile` field and `BucketFile.StorageKey` field — verify exact field names from `internal/domain/bucket/` before writing. If the bucket entity does not carry its files in `ListExpired`, the cleanup must call a separate method or the deletion of storage keys may need to be omitted with a comment. Use what the existing repository interface provides.
